# Game Adjustments

该模组调整游戏内各种机制，模组为娱乐性质。

## NPC 调整

### 防御算法

改动实现在 `GlobalNPCAdjustments.cs` 的 `ModifyIncomingHit` 钩子中。NPC 受到伤害时，模组会先读取其基础防御力并清除原版防御结算，然后将防御力转换为百分比减伤：

```text
减伤比例 = 防御力 / (防御缩放常数 + 防御力)
最终伤害倍率 = 1 - 减伤比例
             = 防御缩放常数 / (防御缩放常数 + 防御力)
```

代码会根据当前游戏难度设置防御缩放常数 `defenseScalingConstant`：

| 游戏难度 | 防御缩放常数 |
| ---: | ---: |
| 普通模式 | 100 |
| 大师模式 | 75 |
| 专家模式 | 50 |

> 防御缩放常数越小，同等防御力提供的减伤越高。难度判断优先匹配大师模式，避免大师模式同时满足专家模式条件时覆盖对应数值。

例如，在没有其他伤害修正且 NPC 防御力为 50 的情况下：

| 游戏难度 | 减伤比例 | 最终承受伤害 |
| ---: | ---: | ---: |
| 普通模式 | 33.33% | 66.67% |
| 大师模式 | 40% | 60% |
| 专家模式 | 50% | 50% |

> 这种算法具有边际收益递减的特点：防御力越高，减伤仍会继续提升，但每一点额外防御带来的提升会逐渐降低，并且正常情况下不会达到 100% 减伤。

> 该钩子属于 `GlobalNPC`，因此改动会应用于游戏中的所有 NPC，包括原版 NPC 和其他模组添加的 NPC。

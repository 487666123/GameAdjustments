# Game Adjustments

该模组调整游戏内各种机制，模组为娱乐性质。

## NPC 调整

### 防御算法

改动实现在 `GlobalNPCAdjustments.cs` 对 `NPC.HitModifiers.GetDamage` 的 IL 修改中。模组移除了护甲穿透结算后防御值不得低于 0 的限制，因此过量护甲穿透可以使原版防御减伤值变为负数。该值随后被转换为百分比修正并应用到 `FinalDamage`，最后将原版固定数值减伤改为 0，避免重复扣除：

```text
减伤比例 = 原版防御减伤值 / (防御缩放常数 + 原版防御减伤值)
最终伤害倍率 = 1 - 减伤比例
             = 防御缩放常数 / (防御缩放常数 + 原版防御减伤值)
```

代码会根据当前游戏难度设置防御缩放常数 `defenseScalingConstant`：

| 游戏难度 | 防御缩放常数 |
| ---: | ---: |
| 普通模式 | 100 |
| 专家模式 | 75 |
| 大师模式 | 50 |

> 防御缩放常数越小，同等原版防御减伤值提供的减伤越高。难度判断优先匹配大师模式，避免大师模式同时满足专家模式条件时覆盖对应数值。

例如，在没有其他伤害修正且原版计算出的防御减伤值为 50 的情况下：

| 游戏难度 | 减伤比例 | 最终承受伤害 |
| ---: | ---: | ---: |
| 普通模式 | 33.33% | 66.67% |
| 专家模式 | 40% | 60% |
| 大师模式 | 50% | 50% |

> 这种算法具有边际收益递减的特点：原版防御减伤值越高，减伤仍会继续提升，但每一点额外减伤带来的提升会逐渐降低，并且正常情况下不会达到 100% 减伤。

当护甲穿透超过 NPC 的有效防御时，原版防御减伤值会变为负数，最终伤害倍率将高于 1，使过量护甲穿透继续提高伤害，而不是在防御降至 0 后失去作用。

> 该公式在原版防御减伤值等于防御缩放常数的负值时分母为 0；超出这一范围也可能产生非正常的伤害倍率。通常游戏数值不会达到该边界，但提供极高护甲穿透的内容可能触发它。

> 该钩子属于 `GlobalNPC`，因此改动会应用于游戏中的所有 NPC，包括原版 NPC 和其他模组添加的 NPC。

## 前缀增强

部分饰品前缀会额外提供通用护甲穿透。穿戴饰品时，效果会自动应用；物品提示也会在前缀提示行后显示绿色的穿甲数值。

| 前缀 | 额外护甲穿透 |
| --- | ---: |
| Precise（精准） | 6 |
| Lucky（幸运） | 10 |
| Jagged（锯齿） | 2 |
| Spiked（尖刺） | 4 |
| Angry（愤怒） | 6 |
| Menacing（险恶） | 8 |

这些穿甲效果通过 `PrefixEnhancement_GlobalItem` 应用于所有带有对应前缀的饰品。

# Game Adjustments

该模组调整游戏内各种机制，模组为娱乐性质。

## 伤害波动

模组通过 `On_Main.DamageVar_float_int_float` 禁用原版伤害波动。伤害计算会直接对传入的伤害值进行四舍五入，不再使用波动百分比和幸运值参数，因此相同条件下的攻击伤害更加稳定。

## NPC 调整

### 防御算法

改动实现在 `DefenseAdjustments_GlobalNPC.cs` 对 `NPC.HitModifiers.GetDamage` 的 IL 修改中。模组移除了护甲穿透结算后防御值不得低于 0 的限制，因此过量护甲穿透可以使原版防御减伤值变为负数。该值随后被转换为百分比修正并应用到 `FinalDamage`，最后将原版固定数值减伤改为 0，避免重复扣除：

```text
减伤比例 = 原版防御减伤值 / (防御缩放常数 + |原版防御减伤值|)
最终伤害倍率 = 1 - 减伤比例
```

代码还会根据当前游戏难度强化 NPC 防御，并设置防御缩放常数 `defenseScalingConstant`：

| 游戏难度 | NPC 防御调整 | 防御缩放常数 |
| ---: | --- | ---: |
| 普通模式 | 不变 | 100 |
| 专家模式 | 基础值 +10，然后整体 ×1.25 | 75 |
| 大师模式 | 基础值 +20，然后整体 ×1.5 | 50 |

> 防御缩放常数越小，同等原版防御减伤值提供的减伤越高。难度判断优先匹配大师模式，避免大师模式同时满足专家模式条件时覆盖对应数值。

例如，在没有其他伤害修正，且经过难度防御强化后，原版计算出的防御减伤值为 50 的情况下：

| 游戏难度 | 减伤比例 | 最终承受伤害 |
| ---: | ---: | ---: |
| 普通模式 | 33.33% | 66.67% |
| 专家模式 | 40% | 60% |
| 大师模式 | 50% | 50% |

> 这种算法具有边际收益递减的特点：原版防御减伤值越高，减伤仍会继续提升，但每一点额外减伤带来的提升会逐渐降低，并且正常情况下不会达到 100% 减伤。

当护甲穿透超过 NPC 的有效防御时，原版防御减伤值会变为负数，最终伤害倍率将高于 1，使过量护甲穿透继续提高伤害，而不是在防御降至 0 后失去作用。由于分母使用减伤值的绝对值，分母始终为正；随着负减伤值的绝对值增大，最终伤害倍率会逐渐接近 2。

> 该钩子属于 `GlobalNPC`，因此改动会应用于游戏中的所有 NPC，包括原版 NPC 和其他模组添加的 NPC。

## 前缀增强

部分饰品前缀会额外提供通用护甲穿透。穿戴饰品时，效果会自动应用；物品提示也会在前缀提示行后显示绿色的穿甲数值。

| 前缀 | 额外护甲穿透 |
| --- | ---: |
| Precise（精准） | 6 |
| Lucky（幸运） | 10 |
| Jagged（锯齿） | 2 |
| Spiked（尖刺） | 4 |
| Angry（愤怒） | 6 |
| Menacing（险恶） | 8 |

这些穿甲效果通过 `PrefixEnhancement_GlobalItem` 应用于所有带有对应前缀的饰品。
